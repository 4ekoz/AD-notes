# what should trigger my attacker my attacker brain if i found WDS in a  network ?  


WDS --> Microsoft service used to **install Windows over the network**. 
Instead of installing Windows with a USB or DVD, companies do this:

1. New computer starts
    
2. It uses **PXE boot** (network boot)
    
3. The computer contacts the **WDS server**
    
4. The server sends a **Windows image**
    
5. Windows installs automatically

So WDS servers store things like:

- Windows installation images
    
- deployment scripts
    
- drivers
    
- sometimes configuration files

A pentester may check WDS for:

### 📂 Windows images

Sometimes images contain:

- old software
    
- credentials
    
- configuration files


Admins often automate installation with scripts like:

unattend.xml

This file can contain:

- local admin passwords
    
- domain join credentials
    
- service account passwords
    

These files are **gold for attackers**.

# What You Actually Need for pentester ? 
WDS = network Windows installation  
✔ uses **PXE boot**  
✔ may store **Windows images and deployment scripts**  
✔ check for **unattend.xml (possible credentials)**

That’s really enough.
