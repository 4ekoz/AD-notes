You’ve successfully created your **Reverse Lookup Zone**! I see you have `1.168.192.in-addr.arpa` set up. This means your DNS server is now ready to handle requests that ask: _"Who owns the IP address 192.168.1.x?"_
# What is "in-addr.arpa"?
It looks like a strange name, but it's just the global standard for Reverse DNS.

- It takes your IP range (**192.168.1**) and writes it **backward**.
    
- It adds `.in-addr.arpa` to tell the computer: _"Look here to turn an IP back into a name."_
Now that the zone exists, it needs **PTR (Pointer) Records**. These are the opposite of the "A Records" in your Forward zone.
