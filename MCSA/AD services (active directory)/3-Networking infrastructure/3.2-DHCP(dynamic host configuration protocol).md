

#  Methods of IP Assignment :

There are four primary ways a PC can obtain an IP address :

1. **Manual (Static):** Manually configured by an administrator.
    
2. **DHCP (Dynamic Host Configuration Protocol):** Automatically assigned (UDP Ports 67 & 68).
    
3. **Alternate Configuration:** A secondary manual IP used if DHCP fails.
    
4. **APIPA (Automatic Private IP Addressing):** A self-assigned address (**169.254.x.x**) used when no DHCP server is found and no alternate is set.


#  How DHCP Works: The DORA Process :


DHCP operates using a four-step broadcast handshake known as **DORA *:

 1. **Discovery:** The client broadcasts a message seeking any available DHCP server.
    
2. **Offer:** The DHCP server responds with an available IP address and configuration.
    
3. **Request:** The client broadcasts back, asking to officially use the offered IP.
    
4. **Acknowledgement (ACK):** The server confirms the assignment and sends the final lease details.

#note : **Lease Time** is the specific duration for which an IP address is temporarily assigned to a device before it must be renewed.



## The Relationship: DHCP, DNS, and AD

DHCP provides the client with **four critical pieces of information**:

1. **IP Address:** The client's identity.
    
2. **Subnet Mask & Gateway:** How to leave the local network.
    
3. **DNS Server Address:** Where the "phonebook" is.
    
4. **Domain Name:** The "Suffix" (e.g., `cita.local`).
    

**The Logic:**  DHCP tells the PC: _"Use this DNS server (e.g., 10.0.0.1)."_

- The PC trusts the DHCP and asks that DNS: _"Where is the Domain Controller? Where is the File Server?"_
    
- **Note:** Active Directory (AD) cannot function without DNS. **Break DNS = Break AD.**
