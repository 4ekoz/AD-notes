# RIp (router information protocol) by rumor
RIP is a very old protocol. It is often called **"Routing by Rumor"** because routers just share their tables with neighbors without knowing the full map of the network.

- **Logic:** It only cares about **Hop Count** (how many routers are in the way).
    
- **The Choice:** RIP would choose the **middle path** (1 → 3) because it’s only **1 hop**.
    
- **The Problem:** RIP is "blind" to speed. It will choose that 56kbps path even though it is extremely slow, simply because it’s the shortest distance.

#  OSPF (open shortest path first )  :
OSPF is much smarter and is the industry standard for most internal networks.

- **Logic:** It uses **Cost**, which is based on **Bandwidth** (speed).
    
- **The Choice:** OSPF would look at the bandwidth.
    
    - The top path (1 → 2 → 3) has **1M**.
        
    - The bottom path (1 → 4 → 5 → 3) has **2M**.
        
    - The middle is a tiny **56kbps**.
        
- **Result:** OSPF would likely choose the **bottom path** (even though it has more hops) because the 2M speed makes it "cheaper" and faster in the long run.

# NAT (network address translation)
![[Pasted image 20260311061107.png]]
types of NAT  :
# 1-Static NAT (one-to-one  mapping) : 
One private IP is permanently mapped to one specific public IP.

- **Use Case:** Hosting a web server inside your network that needs to be reachable from the outside.
# 2-Dynamic NAT : 
The router has a "pool" of public IPs. When a private device needs the internet, the router gives it an available public IP from the pool temporarily.
# 3-PAT (port address translation) (many -to-one mapping):
This is what 99% of people use at home. Hundreds of private IPs share **one** single public IP.

- **How it works:** The router uses **Port Numbers** to distinguish between devices.
    
- **Example:** You have three PCs (**A, B, and C**) in a private network (LAN). All of them are trying to reach a web server at IP `11.0.0.2` on Port `80` ,, hey all happen to be using Source Port `2000` internally ,, They only have **one** public IP address: `11.0.0.1`.
when PC A, B, and C send their data to the switch (**SW**) and then the router ,, The router realizes that if it sends all three requests out using the same public IP and the same port (`2000`), it won't know which reply belongs to whom ,, **The Solution:** The router changes the **Source Port** for each connection as it goes out to the internet (WAN).

# if router don't pass the Broadcast message ? and why ?

- Imagine if every computer in a huge city sent a "shout" (Broadcast) and that shout traveled to every other house in the world. The entire internet would crash in seconds because of the noise.
- A Router's job is to **separate** networks. Each port on a router represents a different **Broadcast Domain**.
- When a PC sends a broadcast (like a DHCP Discover), the Router "stops" it. It doesn't forward it to other networks (WAN or other LANs) to keep the traffic local and clean.
