 

# Forest 
- **Primary DC (Parent):** This is the **Root Domain**. It is the first domain created in the Forest and holds the highest level of authority.
- **Child Domain (e.g., `hr.cita.com`):** A sub-domain that stays under the parent. It has its own database but shares a **Transitive Trust** with the parent.
- **Tree:** When you have multiple sub-domains (like `admins.sales.cita.com`), they form a "Tree" structure.

Additional : 

- Additional domain controller :  This is a second server within the **same domain** (`cita.com`).  ,, If the Primary DC fails, the Additional DC keeps the network running.
- Known as **Replication**. Any user created on one DC is automatically synced to the other.

TRUST : 
- This explains how two completely different organizations talk to each other .
- this is the relationship between `cita.com` and `cisco.com` .
**Scenario:** 
- Imagine "Cita" acquires "Cisco." Instead of migrating thousands of users, you create a **Forest Trust**.
- `cisco.com` can log into a computer in `cita.com` using their original credentials, provided the trust is configured correctly.
