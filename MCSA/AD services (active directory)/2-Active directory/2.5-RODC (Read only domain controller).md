# characteristics of  RoDC : 
1-read only database  -->(like password updates) cannot be made directly on the RODC.
2-**Unidirectional Replication:** Data flows _to_ the RODC, but never _from_ it. This prevents "poisoning".
3-By default, an RODC does not store user passwords. You must explicitly allow specific users (e.g., branch employees) to have their passwords cached locally for offline logins.
