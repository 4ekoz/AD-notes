# what is sam (security account manager) file ? 
- data base stored all users --> too access it from control panel or lacal user and groups 
in windows --> run ->%systemroot%-->system32-->sam 
through this file u can --> reset account administrator , add user , etc .. .
- It is used for "Workgroup" computers or standalone PCs to authenticate users locally.
- **Security:** This is the file that hackers try to "dump" using tools like **Mimikatz** to get local admin passwords.
- once we created AD ,, not sam file is available ,, but we  know that AD  is user repository ,, all data after AD created cut from sam file and set in NTDS (new technology domain services )   file -- > database stored all data in systeml like : users , logs , passwords .. etc 
- **The SAM Still Exists:** Even after you create an AD, the `SAM` file is still there on the hard drive of the Domain Controller.
- **The Shift:** When the server is running as a DC, it **ignores** the SAM file for normal logins. It uses the `NTDS.dit` database instead.
