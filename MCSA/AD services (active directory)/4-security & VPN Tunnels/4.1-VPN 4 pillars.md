- understanding the **4 Pillars of VPN** is crucial. These pillars ensure that when you send data from your Windows 7 client to the Server over an untrusted network (like the internet), it remains secure.

the 4 pillars are the security guards and the reinforced concrete that keep the tunnel safe . 
 
  - # 1- confidentiality .
 This is the most famous pillar. It ensures that if someone (a hacker/ISP) "sniffs" the traffic between your Windows 7 and the Server, they can't read it.

- **How it works:** The data is scrambled using an algorithm (like **AES-256**). Only the two endpoints have the key to unscramble it.
    
- **In your Lab:** If you use Wireshark on the network between your machines without a VPN, you'd see the names of files you transfer. With a VPN, you'll only see "Garbage" (encrypted data).

- # 2-intergrity (hashing)  .
This ensures that the data wasn't **tampered with** during the trip.

- **How it works:** A mathematical "fingerprint" called a **Hash** (like **SHA-256**) is created for the packet. If a hacker changes even one bit of the data (e.g., changing a "Pay $100" command to "Pay $1000"), the hash won't match when it reaches your Windows Server, and the Server will drop the packet.


- # 3-Authenticatoin 
Before the tunnel is built, the two sides must prove they are who they say they are.

- **How it works:** The Windows 7 client must prove its identity to the Server using:
    
    - **Passwords** (User/Pass).
        
    - **Certificates** (Digital IDs).
        
    - **Pre-shared Keys (PSK)**.
        
- **In your Lab:** When you configure the **Routing and Remote Access (RRAS)** role on your Windows Server, you are setting up the "Identity Gatekeeper."

- # 4- anti-replay protection 
This is a sophisticated defense against **Replay Attacks**.

- **How it works:** Each packet is given a unique **Sequence Number**. If an attacker captures a "Login" packet and tries to send it again later to trick the server into logging them in, the Server will see that it has already processed that sequence number and reject it.
    
- **Why it matters:** Even if the attacker can't "read" the encrypted packet, they can still try to "play it back" to repeat an action. This pillar stops that.
